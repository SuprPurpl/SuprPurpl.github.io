External and Internal CSS

So, before I show you how I did the thing with the font,
I'll explain Ex. and Int. CSS.
So Internal CSS, is written in the source-code for one page,
the trick I did with the Font was achieved with Internal CSS use.
What if you want to have a bunch of setting that you can just
copy and paste onto every page you ever make...ever?
Well, that's when you'd use External CSS,
if you've been paying attention you'll'd've (that's right, "you'll'd've" is a real word)
noticed that all of the pages on this site look the same,
that's due to the Style Sheet I've been linking into the <head> of each page.
That style sheet is called "../css/Style1.css" (go find it in the source-code).
That .css file has all the info on what I want the page to look like,
throwing a link into the head of each file is way faster than
typing up EVERY. SINGLE. CHANGE. into EVERY. SINGLE. PAGE.

Groovy, huh?



Examples:




The external link looks like this:

<head> <title>More CSS...</title> <link href="../css/Style1.css" type="text/css" rel="stylesheet" /> </head>
(The link is the bit within the "Link" tag)
I gave you the entire head section too look at so you know Where that link goes.

The internal stuff looks like this:

<head> <title>More CSS...</title> <style type="text/css"> p#Demo { color: rgb(255, 170, 294); font-family: cursive; font-size: 150% } </style> </head>
Once again I gave you the whole head section so you can see what's where,
Now, this is really straight-forward, "p#Demo" is our selector (thing we wanna change)
and if you take a look under it, you can see that I messed with
the color, the kind of font and the size of the font, but that's only half of what went on.
The other half of that change looks like this...

<p id="Demo">Ooh look! this sentence is in a different color and the font is HUGE!?</p>
Note that the <p> tag matches the Selector (aiming it directly at that line)
